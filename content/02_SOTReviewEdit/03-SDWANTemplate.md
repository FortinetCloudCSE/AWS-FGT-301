### SD-WAN Member Section:

The "ul1" IP address is static in this lab.  You will need to create a metadata variable for Underlay 1's gateway in the SD-WAN Member configuration.  The gateway information will be used by any static route that references this SD-WAN member or zone.
Select the WAN1 member, then click the edit button as shown below.
![Edit WAN1](sdwanstaticmember.png?width=950px)

Type $ in the Gateway IP field of the Edit SD-WAN Member window.  This will cause a drop-down list of your available metadata variables to populate.  Click the '+' to create a new metadata variable.
![Edit WAN1 again](sdwaneditwan1.png?width=950px)

Name the new metadata variable ` ul1_gateway_IP `
Click OK.
![Name ul1 gateway](gatewaymeta.png?width=650px)

Select the new matadata variable '$(ul1_gateway_IP)' from the drop-down list in Gateway IP.  Click OK.
![Select ul1 gateway](selectgateway.png?width=650px)

Result:
![Result](wan1result.png?width=950px)

A default static route to SD-WAN that references all of the SD-WAN Zones will be used in this configuration.  This static route will satisfy the route requirement and make all SD-WAN steering rules valid.  A route priority has been set on the Overlay Members to make them less preferred in the routing table.  The will cause any traffic self generated by the FortiGate, such as FortiGuard updates, to use the underlays.  The reason is so that any traffic that falls through to the catch all SD-WAN rule will be sent out the underlays.

### SD-WAN Performance SLA Section:

#### Delete Default Health Checks
The canned Performance SLAs will not be used in this lab.  Delete these unused Performance SLAs to clean up the config and monitor views.  Select all SLAs (except “HUB"), then click the Delete button.

![SLA1](slas.png?width=950px)

#### Create an Internet Health Check
This lab will have a DIA SD-WAN Rule for Branches in the Branches_DIA Device Group. Configure this additional Performance SLA, name it ‘Internet’, and assign it to the Branches_DIA Device Group as an Installation Target.  

 - Start by clicking the + Create New button in the Performance SLA section.
 - Enter the name - ` Internet `
 - Add two server IPs – ` 1.1.1.1 ` and ` 8.8.4.4 `
 - Specify the two underlay members - $(ul1) and $(ul2)
 - Select the "Branches_DIA" device group as the Installation Target
 - Add an SLA Target and define it with the following:
   - Latency = 255ms
   - Jitter = 55ms
   - Packet Loss = 1
 - Toggle both 'Update Static Route' and 'Cascade Interfaces' to off
![Internet HC](inetsla.png?width=950px)


Click to expand the Advanced section:
  - Scroll down to ‘sla-fail-log-period’ and ‘sla-pass-log-period’
  - Set both to 10
  - Click OK
![SLA Logs](slalog.png?width=650px)
Result:
![SLA Result](slas3.png?width=950px)

That completes the edits needed in the SD-WAN Performance SLAs section.  Now let's move to the SD-WAN Rules section.

### SD-WAN Rules Section:
The SOT created the SD-WAN Template without any SD-WAN Rules (the implicit rule is in every SD-WAN Template and is not removable).  You will need to create the steering rules that meet your customer’s needs.  For this lab, we are going to configure 2 basic SD-WAN Rules.

&nbsp;  
&nbsp; 

#### Create a DIA Steering Rule
Start by configuring a new rule called ‘DIA’ for our Branches that will have direct to internet access.  This will steer all non RFC-1918 addresses directly out the underlay interfaces.
 - Click + Create New in the SD-WAN Rules section:

![DIA Rule Part 1 Create](sdwanrule1create.png?width=950px)

 - Fill in the following Fields:
  - Name - DIA
  - Installation Target - Branches_DIA
  - Source Address - all
  - Destination Address - RFC1918-GRP
  - Strategy - Lowest Cost (SLA)
  - Interface Preference:
    - $(ul1)
    - $(ul2)
  - Required SLA Target: Internet#1

![DIA Rule Part 1](sdwanrule1.png?width=950px)
![DIA Rule Part 2](sdwanrule1-22.png?width=950px)
Expand the Advanced Options section.  Find 'dst-negate' and toggle the radio button to on.  Click OK to complete the SD-WAN Steering rule.
![DIA Rule Part 3](sdwanrule1-3.png?width=950px)

Result:
![DIA Result](sdwanrule1-44.png?width=950px)

{{% notice note %}} Setting the selected destination as the “RFC1918-GRP” then selecting “dst-negate” in “Advanced Options” will cause this rule to apply to all traffic EXCEPT the private addresses. In other words, this rule will steer all traffic NOT in RFC1918-GRP which is now an address object group that is configured by default on FMG.
The exclamation point in the red shield shown above indicates the dst-negate configuration option has been enabled.{{% /notice %}}
